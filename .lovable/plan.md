

## תיקון: כתובות עדיין מציגות שכונות סורק מיותרות

### בעיה
הפונקציה `cleanDisplayAddress` מסירה רק חלקים שזהים **בדיוק** לשכונה הקנונית. חלקים כמו "דרום רוטשילד", "כפר שלם", "דרום תל אביב" נשארים כי הם לא שווים ל-"מרכז העיר".

### פתרון פשוט
במקום לנסות לסנן חלקים "רעים" — **לשמור רק את החלק הראשון** (רחוב + מספר) ולהוסיף את השכונה הקנונית. כל מה שמופיע אחרי הפסיק הראשון בכתובת הוא כמעט תמיד שכונה/אזור מהסורק ולא חלק מהכתובת עצמה.

### לוגיקה חדשה (שורות 69-78 ב-`cleanDisplayAddress`)

**לפני:**
```
// Remove neighborhood from comma-separated parts
if (hood) {
  const parts = clean.split(',')...filter(p => p !== hood)
  clean = filtered.join(', ')
}
```

**אחרי:**
```
// Keep only first part (street + number), drop scraper neighborhoods
const parts = clean.split(',').map(p => p.trim()).filter(Boolean);
if (parts.length > 1) {
  clean = parts[0]; // רק רחוב + מספר
}
```
