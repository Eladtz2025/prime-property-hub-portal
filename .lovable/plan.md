

## ניקוי טבלת `street_neighborhoods` + שימוש בה כמקור אמת לשכונות

### מצב נוכחי
- 1,245 שורות, 933 רחובות ייחודיים
- **212 רחובות** עם מספר שכונות (כפילויות) — למשל "אבן גבירול" מופיע 11 פעמים עם שכונות שונות
- שמות השכונות בטבלה הם שמות מפורטים מ-yad2/madlan (כמו "הצפון הישן החלק הצפוני") ולא השמות הקנוניים שלנו ("צפון_ישן")
- 105 שורות עם `neighborhood = NULL`

### תוכנית — 3 שלבים

**שלב 1: מיגרציה — ניקוי הטבלה**

סקריפט SQL שממפה כל שכונה מפורטת לשכונה הקנונית שלנו (מתוך `NEIGHBORHOODS` ב-`locations.ts`):

```sql
-- דוגמאות מיפוי:
'הצפון הישן - צפון', 'הצפון הישן - דרום', 'הצפון הישן החלק הצפוני' → 'צפון ישן'
'הצפון החדש סביבת כיכר המדינה', 'הצפון החדש - כיכר המדינה' → 'כיכר המדינה'
'לב תל אביב', 'לב תל אביב, לב העיר צפון' → 'מרכז העיר'
'דרום רוטשילד', 'שרונה', 'גני שרונה, קרית הממשלה' → 'רוטשילד'
```

אחרי המיפוי — מחיקת כפילויות (שמירה על שורה אחת לכל רחוב):
- אם לרחוב יש שכונה אחת אחרי המיפוי → מושלם
- אם לרחוב עדיין יש 2+ שכונות (רחוב ארוך כמו אבן גבירול) → שמירת **השכונה הנפוצה ביותר** (לפי מספר שורות מקוריות)

**שלב 2: עדכון `cleanDisplayAddress`**

במקום להשתמש ב-`neighborhood` שמגיע מהסורק, הפונקציה:
1. מחלצת שם רחוב מהכתובת (ללא מספר בית)
2. מחפשת ב-cache מקומי של `street_neighborhoods` (נטען פעם אחת)
3. אם נמצא — משתמשת בשכונה מהטבלה
4. אם לא נמצא — fallback לשכונה מהסורק (אחרי ניקוי)

שינוי ב-`ScoutedPropertiesTable.tsx`:
- הוספת query לטעינת `street_neighborhoods` (פעם אחת, ~930 שורות)
- עדכון `cleanDisplayAddress` לקבל את ה-map ולחפש בו

**שלב 3: עדכון שכונות של נכסים קיימים**

מיגרציית נתונים שמעדכנת את עמודת `neighborhood` בטבלת `scouted_properties` לפי הטבלה הנקייה:

```sql
UPDATE scouted_properties sp
SET neighborhood = sn.neighborhood
FROM street_neighborhoods sn
WHERE sp.city = sn.city
  AND /* extract street from sp.address matches sn.street_name */
  AND sp.is_active = true;
```

### סיכום קבצים

| קובץ / פעולה | שינוי |
|---|---|
| Migration SQL | מיפוי שכונות → קנוניות, מחיקת כפילויות, ניקוי NULLs |
| `ScoutedPropertiesTable.tsx` | טעינת street_neighborhoods + עדכון cleanDisplayAddress |
| Migration SQL (data) | עדכון neighborhood בנכסים קיימים |

### הערה חשובה
רחובות ארוכים כמו **אבן גבירול** (חוצה צפון ישן, צפון חדש, כיכר המדינה, מרכז העיר) יקבלו שכונה אחת "ברירת מחדל". אם רוצים דיוק מושלם לפי מספר בית — זה שלב הבא עם טבלת טווחי מספרים.
