
# תיעוד הקוד שפיצח את הסריקות

## מטרה
לתעד את הלוגיקה המותאמת לכל אתר - **הקוד**, לא ההגדרות.

---

## מה הקוד עושה - לפי מקור

### 1. מדלן (scout-madlan) - הפיצוח המרכזי

**ארכיטקטורה: סדרתית**
```text
דף 1 → הצלחה → המתנה 5s → דף 2 → הצלחה → המתנה 5s → דף 3...
דף 1 → חסימה → המתנה 15s (recovery) → דף 2 (ממשיכים!)
```

**למה זה עובד:**
- כל דף מפעיל את הבא (לא מקבילי)
- אם יש חסימה - ממתינים 15 שניות ומנסים את הדף הבא
- לא עוצרים ריצה בגלל חסימה בודדת

**הפרסר (parser-madlan.ts):**
- מטפל ב-2 פורמטים שונים באותו אתר:
  - Format A: מתווכים - בלוקים מפוצלים לשורות
  - Format B: פרטיים - בלוק אחד עם `\\` מפרידים
- מנקה blogs שמופיעים באמצע הדף
- מזהה מתווך לפי patterns: `תיווך`, תמונת סוכן, לינק ל-agents

---

### 2. יד2 (scout-yad2) - הפיצוח

**ארכיטקטורה: מקבילית**
```text
דף 1, 2, 3, 4... → יוצאים בו זמנית עם דילאי 3 שניות בין כל אחד
```

**הפרסר (parser-yad2.ts):**
- מחפש pattern ספציפי: `- [![...](IMG)\\...](url)`
- מזהה מתווכים לפי כפילות שם: `Agency Name\\Agency Name₪`
- מוציא נתונים מתוך **bold text**: `**כתובת, שכונה, עירX חדרים • קומה Y • Z מ״ר**`

---

### 3. הומלס (scout-homeless) - הכי יציב

**ארכיטקטורה: מקבילית**

**הפרסר (parser-homeless.ts):**
- משתמש ב-cheerio לקריאת HTML
- מחפש שורות טבלה: `tr[type="ad"]`
- קורא עמודות לפי סדר קבוע:
  - עמודה 3: עיר
  - עמודה 4: שכונה
  - עמודה 5: רחוב
  - עמודה 6: חדרים
  - עמודה 7: קומה
  - עמודה 8: מחיר
- **תמיד מסומן כפרטי** (הומלס זה בעיקר מודעות פרטיות)

## URL Building - הלוגיקה השונה

**מדלן:**
```
https://www.madlan.co.il/for-rent/תל-אביב-יפו-ישראל?page=2
```

**יד2:**
```
https://www.yad2.co.il/realestate/rent?topArea=2&area=1&city=5000&page=2
```

**הומלס:**
```
https://www.homeless.co.il/rent/?inumber1=17,1,150&page=2
```

## הניקוי החכם (לפני הפרסינג)

כל אתר עם לוגיקת ניקוי משלו:

| אתר | מה מנקים |
|-----|----------|
| מדלן | ניווט בהתחלה, blogs באמצע, footer |
| יד2 | headers, פרויקטים חדשים, footer |
| הומלס | ניווט לפני "כרגע בלוח", pagination אחרי |
