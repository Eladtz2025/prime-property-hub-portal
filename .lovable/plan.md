

# הוספת "פרויקט מעקב" כאפשרות נפרדת בתפריט סוג נכס

## מה חסר

כרגע יש בתפריט רק "פרויקט חדש" -- וה-URL למעקב מוסתר כשדה אופציונלי בתוך הטופס. אתה רוצה שיהיה **ברור ומופרד**: אפשרות אחת להזנה ידנית, ואפשרות שנייה למעקב אוטומטי מ-URL.

## הפתרון

הוספת אפשרות 5 בתפריט סוג הנכס:

| ערך | שם בתפריט | מה מוצג בטופס |
|-----|-----------|---------------|
| rental | השכרה | שדות רגילים |
| sale | מכירה | שדות מכירה |
| management | ניהול נכסים | שדות ניהול |
| project | פרויקט חדש | שדות פרויקט ידניים (כמו היום) |
| **project** | **פרויקט מעקב** | **URL נדרש + שם + עיר בלבד, שאר הנתונים מהסריקה** |

**חשוב:** שניהם נשמרים בדאטאבייס כ-`property_type = 'project'`. ההבדל הוא רק ב-UI -- "פרויקט מעקב" דורש URL ומציג טופס מצומצם כי הנתונים יגיעו מהסריקה.

## שינויים

### קובץ 1: `src/components/AddPropertyModal.tsx`

**תפריט סוג נכס** -- הוספת אפשרות חמישית:
```text
<SelectItem value="rental">השכרה</SelectItem>
<SelectItem value="sale">מכירה</SelectItem>
<SelectItem value="management">ניהול נכסים</SelectItem>
<SelectItem value="project">פרויקט חדש</SelectItem>
<SelectItem value="tracked_project">פרויקט מעקב</SelectItem>
```

**state נוסף** -- `formMode` (פנימי, לא נשמר ל-DB):
- כש-`property_type = 'project'` -- טופס ידני מלא (כמו היום)
- כש-`property_type = 'tracked_project'` -- טופס מצומצם:
  - URL מעקב (חובה, בולט למעלה)
  - שם הפרויקט (כותרת)
  - עיר
  - הערות (אופציונלי)
  - **בלי**: טווח חדרים, טווח שטח, יח"ד, סטטוס, בעלים, שוכר -- כי זה יגיע מהסריקה

**בזמן שמירה** -- המרה ל-`property_type = 'project'` לפני שליחה ל-DB, עם trackingUrl מלא.

### קובץ 2: `src/components/PropertyEditRow.tsx`

אין שינוי -- בעריכה כבר מוצג שדה tracking URL לפרויקטים, וזה מספיק.

### ללא שינויי DB

הכל נשמר כ-`property_type = 'project'` עם/בלי `tracking_url`. אין צורך בעמודות או טבלאות חדשות.

## איך הטופס ייראה ל"פרויקט מעקב"

```text
+---------------------------------------+
| סוג נכס: [פרויקט מעקב ▼]   עיר: [___]|
+---------------------------------------+
| 🔗 קישור לעמוד הפרויקט (חובה)        |
| [https://www.example.co.il/project/.. ]|
| המערכת תסרוק את העמוד ותחלץ את הדירות  |
+---------------------------------------+
| שם הפרויקט                            |
| [___________________________________ ]|
+---------------------------------------+
| הערות                                 |
| [___________________________________ ]|
+---------------------------------------+
|          [ הוסף פרויקט מעקב ]          |
+---------------------------------------+
```

## סיכום

- קובץ אחד לעדכון: `AddPropertyModal.tsx`
- הוספת "פרויקט מעקב" כאפשרות בתפריט
- טופס מצומצם: URL (חובה) + שם + עיר + הערות
- שמירה כ-`property_type = 'project'` עם `tracking_url`
- הפרויקט הידני נשאר בדיוק כמו שהוא
