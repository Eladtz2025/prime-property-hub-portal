
# תוכנית תיקון: זיהוי תיווך/פרטי ביד2

## הבעיה שנמצאה

הפארסר לא מזהה נכון מי תיווך ומי פרטי ביד2. הנכס שנשמר כ"פרטי" (שלומציון המלכה, 13,000 שח) הוא בעצם של משרד תיווך - בעמוד המודעה כתוב "נכסים נוספים של המשרד" עם לינק לסוכנות.

**שורש הבעיה**: הלוגיקה הנוכחית מחפשת מילות מפתח כמו "תיווך", "רימקס", מספר רישיון 7 ספרות וכו'. אבל ביד2, מודעות תיווך פשוט מציגות את שם הסוכנות (למשל "הוריזן מרקטינג", "סוקולוב נכסים") **בלי** מילות מפתח חזקות. כשאין התאמה, המערכת מניחה שזה פרטי - וזה שגוי.

## הפתרון: זיהוי מבוסס מבנה (Structural Detection)

המבנה של הבלוק ב-SERP שונה בין תיווך לפרטי:

```text
+-- מודעת תיווך --+         +-- מודעה פרטית --+
|                   |         |                  |
| [![כתובת](IMG)\\  |         | [![כתובת](IMG)\\ |
| \\                |         | \\               |
| שם סוכנות\\      |  <-- אין שורה כזו -->      |
| \\                |         |                  |
| שם סוכנות₪ מחיר |         | ₪ מחיר           |
| \\                |         | \\               |
| **פרטים...**     |         | **פרטים...**     |
+-------------------+         +------------------+
```

במודעת תיווך: שם הסוכנות מופיע פעמיים לפני סימן ₪
במודעה פרטית: הסימן ₪ מגיע ישר אחרי התמונה (בלי טקסט ביניהם)

## שינויים טכניים

### 1. עדכון `parseYad2Block` ב-`parser-yad2.ts`

שינוי הלוגיקה של זיהוי broker/private:

- **בדיקה מבנית חדשה**: לבדוק את הטקסט בין סוף ה-image markdown (`]`) לבין סימן ₪. אם יש טקסט שאינו רק סלאשים/רווחים/שורות ריקות -- זה שם סוכנות = תיווך.
- **שמירת שם הסוכנות**: לחלץ את שם הסוכנות ולשמור אותו (לצורך debug ובעתיד כשדה).
- **הסרת הפאלבק** שמניח שכל מה שלא זוהה הוא "פרטי" -- במקום זה, null (לא ידוע).

הלוגיקה החדשה:
1. חלץ את הטקסט בין סוף `](IMG)` לבין `₪`
2. נקה אותו מ-backslashes, RTL markers, רווחים
3. אם נשאר טקסט (שהוא לא כתובת ה-alt) -> תיווך
4. אם אין טקסט -> בדוק תגיות אחרי ה-bold (מפרטי/ללא תיווך) -> פרטי
5. אחרת -> null (לא ידוע, הbackfill יבדוק מאוחר יותר)

### 2. שמירת ה-raw_text בפועל

כרגע `raw_text` נשמר כ-null בדאטאבייס למרות שהפארסר שולח אותו. צריך לוודא שה-`saveProperty` שומר את `raw_text` כ-`raw_data` כדי שנוכל לנפות בעיות בעתיד.

### 3. בדיקות

- להריץ סריקה חדשה עם הלוגיקה המתוקנת
- לבדוק שמודעות תיווך מזוהות כתיווך ופרטיות כפרטי
- לוודא שהפילטר `owner_type_filter: 'private'` אכן מסנן נכון
