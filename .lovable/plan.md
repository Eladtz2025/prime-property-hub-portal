

# הוספת סינון שכונות לקונפיגורציות הסקאוט

## ממצאים מהחקירה

### מבנה URL לכל מקור:

**Yad2** - פרמטר `neighborhood` עם קודים מספריים (תומך במספר שכונות):
```
https://www.yad2.co.il/realestate/rent?city=5000&neighborhood=1483,1518
```

**Madlan** - שם השכונה בנתיב ה-URL (לפני העיר):
```
https://www.madlan.co.il/for-rent/בבלי-תל-אביב-יפו-ישראל
```
(הערה: תומך בשכונה אחת בלבד ב-URL)

**Homeless** - פרמטר `inumber1` עם קודי אזורים:
```
https://www.homeless.co.il/rent/?inumber1=1   (תל אביב צפון)
https://www.homeless.co.il/rent/?inumber1=2   (תל אביב דרום)
https://www.homeless.co.il/rent/?inumber1=382 (תל אביב מזרח)
```

### מה כבר קיים במערכת:
- עמודת `neighborhoods` (text array) בטבלת `scout_configs` - קיימת
- קומפוננטת `NeighborhoodSelectorDropdown` - קיימת ומוכנה
- קודי שכונות Yad2 קיימים בארכיון (`supabase/functions/_archived/_personal-scout/neighborhood-codes.ts`)
- רשימת שכונות לפי ערים ב-`src/config/locations.ts`

### 2. עדכון ה-UI בטופס עריכת קונפיגורציה
**קובץ:** `src/components/scout/UnifiedScoutSettings.tsx`

שינויים:
- הוספת `neighborhoods: []` ל-state של `formData` (שורה 156)
- import של `NeighborhoodSelectorDropdown` מהקומפוננטה הקיימת
- הוספת שדה בחירת שכונות מתחת לשדה העיר (אחרי שורה 816)
- עדכון `openEditDialog` לטעון שכונות קיימות (שורה 488)
- עדכון `handleSubmit` לשמור שכונות (שורה 515)
- עדכון ה-mutations לכלול `neighborhoods` בנתונים
- הצגת badge עם מספר השכונות בכרטיס הקונפיגורציה

### 3. עדכון URL Builders
**קובץ:** `supabase/functions/_shared/url-builders.ts`

שינויים:
- import קודי שכונות מהקובץ החדש
- עדכון `buildSinglePageUrl`:
  - Yad2: הוספת פרמטר `neighborhood=code1,code2,...` (שורה ~120)
  - Madlan: החלפת ה-slug של העיר ב-slug של שכונה (שורה ~145)
  - Homeless: הוספת פרמטר `inumber1` לפי קוד האזור (שורה ~160)
- עדכון `buildSearchUrls` באותו אופן

## אסטרטגיה לכל מקור

### Yad2 (תמיכה מלאה)
- מספר שכונות ב-URL אחד: `&neighborhood=1483,1518,205`
- סינון ברמת השרת

### Madlan (שכונה אחת)
- אם נבחרה **שכונה אחת**: URL עם השכונה
- אם נבחרו **מספר שכונות**: סריקת העיר כולה + סינון בשלב ה-parsing
- הודעת warning ב-UI

### Homeless (אזורים)
- סינון לפי אזור (צפון/דרום/מזרח) עם `inumber1`
- שכונות באותו אזור מאוחדות לקוד אחד
- אם נבחרו שכונות ממספר אזורים - ריצות נפרדות או סינון post-scan

## בדיקות לאחר היישום

1. יצירת קונפיגורציית **Yad2** עם שכונות - בדיקה שה-URL נבנה עם `&neighborhood=...`
2. יצירת קונפיגורציית **Madlan** עם שכונה אחת - בדיקה שהשכונה מופיעה ב-URL
3. יצירת קונפיגורציית **Homeless** עם שכונות - בדיקה שה-`inumber1` נכון
4. עריכת קונפיגורציה קיימת והוספת שכונות - בדיקה שהשכונות נשמרות
5. הרצת סריקה עם שכונות - בדיקה שהדירות שחוזרות מהשכונות הנכונות
