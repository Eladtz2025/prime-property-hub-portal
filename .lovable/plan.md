
# תיקון חילוץ מחירים - Format D (Jina)

## הבעיה

המחירים בפורמט Jina (שורה אחת) עדיין מתמזגים עם מספר החדרים:
- `80,000,001` = `8,000,000` + `1` (חדר)
- `71,000,004` = `7,100,000` + `4` (חדרים)
- `45,000,009` = `4,500,000` + `9` (חדרים)

הסיבה: הרג'קס מוצא את ה-₪ אבל הספרות של מספר החדרים צמודות למחיר בטקסט הגולמי, גם אחרי ניקוי RTL marks. הפורמט הגולמי הוא כנראה:
```
8,000,000₪ 1 חד׳
```
אבל אחרי ניקוי ה-RTL, הרג'קס `/([\d,]+)\s*₪/` תופס `80000001₪` כי אין רווח אמיתי בין ה-₪ למספר.

## הפתרון

### קובץ: `supabase/functions/_experimental/parser-madlan.ts`

#### גישה: חילוץ מחיר עם ולידציה הגיונית

במקום להסתמך רק על רג'קס, נוסיף ולידציה:

1. **חילוץ הפוך**: נחפש `₪` ואז נלך אחורה לקחת רק ספרות ופסיקים -- לא קדימה
2. **ולידציה**: אם מחיר מכירה > 50,000,000 (50 מיליון), כנראה שיש ספרות מודבקות -- ננסה לחתוך
3. **רג'קס משופר**: `/([\d,]+)\s*₪/` יוחלף ב-`/([\d]{1,3}(?:,\d{3})*)\s*₪/` שתופס רק מספרים בפורמט נכון (עם פסיקים כל 3 ספרות)

#### גישה חלופית (עדיפה): חילוץ חכם מהקונטקסט

במקום רג'קס גנרי, נחפש את הדפוס הספציפי של מדלן:
```
PRICE₪ ROOMS חד׳
```
ונשתמש ברג'קס שתופס את שני השדות בבת אחת:
```typescript
const priceRoomsMatch = cleanedBlock.match(/([\d,]+)\s*₪\s*([\d.]+)\s*חד[׳']/);
```
כך נדע בדיוק איפה נגמר המחיר ואיפה מתחיל מספר החדרים.
