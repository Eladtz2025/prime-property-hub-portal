

# תיקון פורמט URL של שכונות מדלן

## הבעיה שזיהית

מדלן מפרש את "הצפון הישן" כ**רחוב** ולא כשכונה!

| מה רשמנו | מה מדלן רואה | תוצאה |
|----------|--------------|-------|
| `הצפון-הישן-תל-אביב-יפו-ישראל` | רחוב | 0 דירות |
| `שכונה-הצפון-הישן-החלק-הצפוני-תל-אביב-יפו-ישראל` | שכונה | 72 דירות |

## הפתרון

### שינוי 1: עדכון מיפוי השכונות למדלן

**צפון ישן** - צריך להפוך ל-3 תת-אזורים:
- `שכונה-הצפון-הישן-החלק-הצפוני`
- `שכונה-הצפון-הישן-החלק-המרכזי`
- `שכונה-הצפון-הישן-החלק-הדרום-מזרחי`

**צפון חדש** - צריך להפוך ל-3 תת-אזורים:
- `שכונה-הצפון-החדש-החלק-הצפוני`
- `שכונה-הצפון-החדש-החלק-הדרומי`
- `שכונה-הצפון-החדש-סביבת-כיכר-המדינה`

### שינוי 2: מבנה מיפוי חדש

במקום slug בודד לכל שכונה, צריך **מערך של slugs**:

```typescript
// מבנה חדש: שכונה -> מערך תת-אזורים
export const madlanNeighborhoodSubAreas: Record<string, string[]> = {
  'צפון_ישן': [
    'שכונה-הצפון-הישן-החלק-הצפוני',
    'שכונה-הצפון-הישן-החלק-המרכזי',
    'שכונה-הצפון-הישן-החלק-הדרום-מזרחי',
  ],
  'צפון_חדש': [
    'שכונה-הצפון-החדש-החלק-הצפוני',
    'שכונה-הצפון-החדש-החלק-הדרומי',
    'שכונה-הצפון-החדש-סביבת-כיכר-המדינה',
  ],
  // שכונות אחרות עם תת-אזור בודד
  'בבלי': ['שכונה-בבלי'],
  'נווה_צדק': ['שכונה-נווה-צדק'],
  // ...
};
```

### שינוי 3: עדכון הפונקציה getMadlanMultiNeighborhoodPath

```typescript
export function getMadlanMultiNeighborhoodPath(
  neighborhoods: string[], 
  city: string
): string | null {
  const allSlugs: string[] = [];
  const citySlug = madlanCityMap[city] || city.replace(/\s+/g, '-');
  
  for (const neighborhood of neighborhoods) {
    // NEW: Get array of sub-areas instead of single slug
    const subAreas = madlanNeighborhoodSubAreas[neighborhood] || 
                     madlanNeighborhoodSubAreas[neighborhood.replace(/ /g, '_')];
    
    if (subAreas) {
      for (const subArea of subAreas) {
        // Format: שכונה-שם-תת-אזור-city-ישראל
        allSlugs.push(`${subArea}-${citySlug}-ישראל`);
      }
    }
  }
  
  if (allSlugs.length === 0) return null;
  
  return allSlugs.join(',');
}
```
