

## מיזוג חד-פעמי/אוטומטי לשדה רגיל במקום טאבים

### הבעיה
כרגע "חד-פעמי" ו"אוטומטי חוזר" מוצגים ככפתורי toggle (כמו טאבים) בראש הטופס, כאילו הם שני מצבים שונים. בפועל רוב השדות זהים — הבדל בין השניים הוא רק שדה "תדירות" (חד-פעמי = פעם אחת, אוטומטי = כל X ימים).

### הפתרון
הסרת כפתורי ה-toggle והפיכת "סוג פרסום" לשדה Select רגיל בשורה הראשונה של הטופס, ליד שאר השדות:

```text
┌─────────────────────────────────────────────────────┐
│ סוג פרסום: [חד-פעמי ▾]   מקור: [בחר נכס ▾]        │
│            [אוטומטי חוזר]  תבנית: [בחר תבנית ▾]    │
│            [כתבות שבועי]                             │
│                                                      │
│ פלטפורמה: [Facebook] [Instagram]                    │
│ ...                                                  │
└─────────────────────────────────────────────────────┘
```

### שינויים ב-`AutoPublishManager.tsx`

1. **הסרת כפתורי toggle** (שורות 382-401) — הכפתורים "חד-פעמי" ו"אוטומטי חוזר"
2. **שדה Select חדש "סוג פרסום"** — עם 3 אפשרויות:
   - "חד-פעמי" (one_time) — פרסום מיידי/מתוזמן
   - "אוטומטי — דירות" (recurring + property_rotation) — סבב דירות מחזורי
   - "אוטומטי — כתבות" (recurring + article_oneshot) — כתבות חד-פעמיות
3. **מיזוג שדות שורה ראשונה** — Select "סוג פרסום" + מקור/נכס + תבנית באותה שורה grid
4. **שדות מותנים** — תדירות/שעה/סוג נכס מופיעים רק כש-mode=recurring (כמו היום)
5. **הסרת ה-Select "סוג" הנפרד** (שורות 411-427) — כי הוא מתמזג לתוך "סוג פרסום"

**קובץ אחד, ~30 שורות שינוי.**
