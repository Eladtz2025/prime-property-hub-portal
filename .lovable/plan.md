

## מיזוג מלא — מערכת פרסום אחת ללא פופאפים וללא הפרדה

### הרעיון
במקום שתי מערכות (אוטומטית עם Dialog + ידנית), ייבנה **רכיב אחד** שמציג הכל inline על המסך. המשתמש יוצר "תבנית פרסום" ובוחר את **סוג הפרסום**:

| סוג | תיאור |
|-----|-------|
| חד-פעמי | פרסום מיידי או מתוזמן לתאריך ספציפי |
| חוזר אוטומטי | כל X ימים, המערכת מפרסמת לבד (round-robin על דירות) |

### מה משתנה

**`AutoPublishManager.tsx` — שכתוב מלא:**

1. **הסרת Dialog** — טופס יצירת תבנית חדשה מוצג inline על הדף (כמו הקומפוזר הנוכחי), לא בפופאפ
2. **הסרת SocialPostComposer** — במקום embed שלו, הטופס האחיד משמש גם ליצירת תבנית אוטומטית וגם לפרסום חד-פעמי
3. **בחירת מצב פרסום** — שורת כפתורים/toggle בראש הטופס:
   - "חד-פעמי" → מציג בחירת נכס + תאריך/שעה או פרסום מיידי (כמו הקומפוזר היום)
   - "אוטומטי" → מציג בחירת תדירות + שעה + תבנית עם placeholders (כמו הדיאלוג היום)
4. **כרטיסי תבניות קיימות** — נשארים למטה, קומפקטיים, עם switch/edit/delete
5. **כפתור "צור תבנית חדשה +"** — פותח את הטופס inline (לא פופאפ), עם אנימציית expand

### מבנה הטופס האחיד (inline)

```text
┌─────────────────────────────────────────┐
│ [חד-פעמי] [אוטומטי חוזר]    ← toggle  │
│                                         │
│ שם התבנית: [___________]               │
│ פלטפורמות: [Facebook] [Instagram]       │
│                                         │
│ אם חד-פעמי:                            │
│   בחר נכס: [dropdown]                  │
│   תאריך: [calendar] שעה: [10:00]       │
│                                         │
│ אם אוטומטי:                            │
│   תדירות: [יומי ▾]  שעה: [10:00]      │
│   תבנית פוסט: [textarea + placeholders] │
│                                         │
│ האשטגים: [chips selector]              │
│ תמונות: [grid]                         │
│                                         │
│ [שמור תבנית] / [פרסם עכשיו]           │
└─────────────────────────────────────────┘

── תבניות קיימות ──
[כרטיס 1] [כרטיס 2] ...

── היסטוריה ──
```

### שינויים טכניים

| קובץ | שינוי |
|-------|-------|
| `AutoPublishManager.tsx` | שכתוב — הסרת Dialog, טופס inline עם toggle חד-פעמי/אוטומטי, שילוב לוגיקת הקומפוזר (בחירת נכס, פרסום מיידי, תזמון) |
| `SocialDashboard.tsx` | ללא שינוי — כבר מציג רק AutoPublishManager |
| `SocialPostComposer.tsx` | לא נמחק, אבל כבר לא בשימוש מ-Dashboard (האלמנטים שלו עוברים ל-AutoPublishManager) |

**קובץ עיקרי אחד, ~250 שורות שכתוב.**
