
# תוכנית: תיקון זיהוי לינקים שבורים ב-Yad2

## הבעיה שזוהתה

הבדיקה היומית של לינקים (`trigger-availability-check-daily`) רצה כל יום אבל **לא מצליחה לזהות לינקים שבורים** ב-Yad2.

### הסיבה הטכנית
```text
┌─────────────────────────────────────────────────────────────────┐
│                      הבעיה הנוכחית                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Edge Function                        Yad2 Server               │
│   ┌──────────────┐                    ┌──────────────┐          │
│   │ fetch()      │ ──────────────────>│ Bot detected │          │
│   │ User-Agent:  │                    │              │          │
│   │ PropertyCheck│                    │ Returns:     │          │
│   └──────────────┘                    │ Status 200   │          │
│         │                             │ But DIFFERENT│          │
│         │                             │ HTML content │          │
│         ▼                             └──────────────┘          │
│   ❌ לא מוצא את                                                  │
│   "חיפשנו בכל מקום"                                              │
│   כי Yad2 מחזיר HTML                                             │
│   אחר לבוטים                                                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

- הקוד הנוכחי משתמש ב-`fetch()` ישיר עם User-Agent פשוט
- Yad2 מזהה את הבקשה כבוט ומחזיר תוכן שונה (סטטוס 200 אבל בלי ההודעות העבריות)
- המחרוזות לזיהוי ("חיפשנו בכל מקום", "המודעה הוסרה") לא מופיעות ב-HTML שהבוט מקבל
- הפונקציות Scout משתמשות ב-**Firecrawl עם stealth proxy** ומצליחות לקבל את התוכן האמיתי

### הנתונים
| מדד | ערך |
|-----|-----|
| נכסים פעילים | 5,889 |
| נכסים לא פעילים | 17 בלבד |

## הפתרון המוצע

שימוש ב-**Firecrawl** (כבר מחובר ועובד!) במקום `fetch()` ישיר.

```text
┌─────────────────────────────────────────────────────────────────┐
│                      הפתרון החדש                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Edge Function                        Firecrawl                 │
│   ┌──────────────┐                    ┌──────────────┐          │
│   │ Firecrawl    │ ──────────────────>│ Stealth Proxy│          │
│   │ API call     │                    │ + Real browser│          │
│   └──────────────┘                    │ headers      │          │
│         │                             └──────┬───────┘          │
│         │                                    │                  │
│         │                                    ▼                  │
│         │                             ┌──────────────┐          │
│         │                             │ Yad2 Server  │          │
│         │                             │ (thinks it's │          │
│         │                             │  real user)  │          │
│         │                             └──────┬───────┘          │
│         │                                    │                  │
│         ▼                                    ▼                  │
│   ✅ מקבל את התוכן                    Returns real HTML         │
│   האמיתי עם "חיפשנו                   with "חיפשנו בכל מקום"    │
│   בכל מקום אבל אין                    visible                   │
│   לנו עמוד כזה"                                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## שינויים טכניים

### קובץ: `supabase/functions/check-property-availability/index.ts`

**שינוי 1:** הוספת import ל-scraping utilities
```typescript
import { scrapeWithRetry, validateScrapedContent } from "../_shared/scraping.ts";
```

**שינוי 2:** הוספת קריאה ל-Firecrawl API key
```typescript
const firecrawlApiKey = Deno.env.get('FIRECRAWL_API_KEY');
```

**שינוי 3:** החלפת הלוגיקה הקיימת (שורות 105-182) בלוגיקה חדשה

הלוגיקה החדשה:
1. עבור **Yad2** ו-**Madlan** - משתמש ב-Firecrawl עם stealth/auto proxy
2. עבור **Homeless** ואחרים - משתמש ב-fetch ישיר (פחות מגבלות)
3. בודק את ה-Markdown המוחזר עבור indicators של מודעה שהוסרה
4. בודק גם אם ה-scrape נכשל לחלוטין (סימן שהדף לא קיים)

**שינוי 4:** הוספת מחרוזות זיהוי נוספות
```typescript
// Yad2 specific - content patterns
'חיפשנו בכל מקום',          // Shortened version
'אופס',                      // Error page
'הדף המבוקש לא נמצא',        // Page not found
'לא הצלחנו למצוא',           // Couldn't find

// Madlan specific
'הנכס לא נמצא',
'הדירה אינה זמינה',
```

**שינוי 5:** שימוש בהגדרות timeout ארוכות יותר
- Firecrawl צריך יותר זמן (כולל rendering)
- שינוי `head_timeout_ms` ל-15 שניות
- שינוי `get_timeout_ms` ל-12 שניות

## עלויות Firecrawl

| סוג בקשה | קרדיטים |
|----------|---------|
| Yad2 (stealth proxy) | 5 קרדיטים לבקשה |
| Madlan (auto proxy) | 1 קרדיט לבקשה |
| Homeless (fetch ישיר) | 0 קרדיטים |

### הערכת עלות יומית
- Yad2: ~4,000 נכסים × 5 = ~20,000 קרדיטים
- Madlan: ~1,500 נכסים × 1 = ~1,500 קרדיטים
- **סה"כ יומי: ~21,500 קרדיטים**

**המלצה:** להוסיף הגדרה `use_firecrawl_for_availability` ב-scout_settings שתאפשר לכבות את זה אם העלות גבוהה מדי.

## אופטימיזציה: בדיקה חכמה

כדי לחסוך קרדיטים, נבדוק רק נכסים "חשודים":

1. **עדיפות 1:** נכסים ישנים מ-14 יום (סבירות גבוהה שהוסרו)
2. **עדיפות 2:** נכסים שלא נבדקו מעולם
3. **עדיפות 3:** נכסים שהבדיקה הקודמת נכשלה (timeout/error)

```typescript
// New setting in scout_settings
availability_check_mode: 'smart' | 'full'
```

**מצב smart:**
- בודק עד 500 נכסים ביום
- מתעדף נכסים ישנים
- חוסך ~90% מהקרדיטים

**מצב full:**
- בודק את כל הנכסים
- יקר יותר
- מומלץ להרצה שבועית

## שאלה לפני יישום

האם להפעיל **מצב smart** (500 נכסים ביום, חוסך קרדיטים) או **מצב full** (כל הנכסים כל יום)?

המלצה: להתחיל עם **full** לניקוי ראשוני, ואז לעבור ל-**smart** לתחזוקה שוטפת.
