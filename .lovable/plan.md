

# תיקון פילטר שכונות - Property Scout

## הבעיה

הפילטר לפי שכונות לא עובד כי יש אי-התאמה בין:
1. **הגדרות הקבוצות** (`NEIGHBORHOOD_GROUPS`) בקוד
2. **שמות השכונות בפועל** שנשמרים ב-database

### דוגמאות לאי-התאמה

| קבוצה בקוד | דפוס מוגדר | שם ב-DB | סטטוס |
|------------|-------------|---------|-------|
| צפון ישן | `הצפון הישן` | `צפון ישן` (716 נכסים!) | לא נתפס |
| צפון ישן | `הצפון הישן - צפון` | `הצפון הישן - צפון` | עובד |
| לב העיר | `לב העיר`, `מרכז העיר` | `לב תל אביב, לב העיר צפון` (197) | לא נתפס |
| צפון חדש | `כיכר המדינה` | `הצפון החדש סביבת כיכר המדינה` (66) | לא נתפס |
| רמת אביב | `הגוש הגדול` (קבוצה נפרדת) | `הגוש הגדול, רמת אביב החדשה, נופי ים` (60) | לא נתפס |

### שלב 2: שיפור לוגיקת החיפוש

במקום partial matching פשוט, נשתמש ב-matching מדויק יותר:

```typescript
// בפונקציה applyFilters
if (filters.neighborhoods.length > 0) {
  const allPatterns: string[] = [];
  filters.neighborhoods.forEach(n => {
    const patterns = NEIGHBORHOOD_GROUPS[n] || [n];
    allPatterns.push(...patterns);
  });
  
  // שימוש ב-OR עם patterns שמחפשים התאמה מדויקת יותר
  const orConditions = allPatterns.map(p => {
    // אם הדפוס מכיל פסיק - חיפוש כתת-מחרוזת
    // אחרת - חיפוש מדויק יותר
    return `neighborhood.ilike.%${p}%`;
  }).join(',');
  
  query = query.or(orConditions);
}
```
